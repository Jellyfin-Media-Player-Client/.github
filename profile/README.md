## Why This Exists

A Jellyfin server organizes your collection; the browser tab is not always the best way to watch it on a TV or HTPC. **Jellyfin Media Player** exists to give that server a native, remote-friendly front end that feels closer to appliance software than to a desktop browser chrome.

**Jellyfin Media Player** solves **one specific problem**: playing media from *your* Jellyfin instance with desktop performance and ten-foot navigation. No third-party catalog. No rented shelves. Just the libraries you already host on disks you control.

If you run Jellyfin for family rips, home videos, or music and want a dedicated client, **Jellyfin Media Player** is for you. It keeps continue-watching, user profiles, and library shelves synchronized with the server API while you focus on the remote and the screen.

## Understanding Jellyfin Media Player Core Components

**Jellyfin Media Player** is not a replacement for the server. It combines several client-side pieces:

- **Server connection** — Talks to your Jellyfin instance over the LAN or a secure remote URL.
- **Library views** — Renders movies, series, and music shelves generated by Jellyfin metadata.
- **Playback engine** — Uses local decode paths where available so **Jellyfin Media Player** stays smooth on TV hardware.
- **User sessions** — Respects Jellyfin permissions, parental settings, and continue-watching state.
- **Remote-friendly UI** — Larger focus targets for controllers and HTPC remotes.
- **Subtitle and audio controls** — Switch tracks mid-playback without leaving the client.
- **Fullscreen living-room mode** — Hides desktop clutter so **Jellyfin Media Player** feels like appliance software.

All these work together seamlessly with your Jellyfin server. Index once on the host; enjoy everywhere the client runs.

When relatives visit, they pick a profile, see only the shelves they should, and leave watched progress where it belongs—without learning your server’s admin screens.
